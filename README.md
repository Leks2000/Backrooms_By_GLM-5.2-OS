# Backrooms — Уровень B2

Хоррор-игра от первого лица в стиле Backrooms. Полностью самодостаточный один HTML-файл без внешних библиотек. Рендер на основе raycaster (DDA) с текстурированными стенами и потелочным/напольным кастингом по строкам. Бесконечный процедурный мир на чанках 16×16, A* pathfinding, конечный автомат монстра, процедурные текстуры, Web Audio API, постобработка.

## Запуск

Открыть `index.html` в браузере. Игра требует мышь и клавиатуру. Для захвата курсора — клик по экрану. Рекомендуется Chrome/Firefox с поддержкой Pointer Lock и Web Audio API.

## Управление

| Клавиша | Действие |
|---------|----------|
| W A S D | Движение |
| SHIFT   | Бег (тратит выносливость) |
| CTRL    | Присесть (тише, медленнее) |
| E       | Взаимодействие (двери, генератор, лифт) |
| F       | Фонарь (вкл/выкл) |
| H       | Принять таблетки (+25 HP) |
| R       | Начать заново (после смерти) |
| ESC     | Пауза / возобновление |
| Мышь    | Осмотр |

## Цель игры

1. **Найти 8 документов** — разбросаны по миру, содержат лор и триггеры, меняющие механики (монстр начинает реагировать на фонарь/бег/двери/красные двери).
2. **Собрать 3 детали генератора** — необходимы для восстановления питания.
3. **Включить генератор** — восстанавливает освещение (мир становится ярче, двери можно активировать).
4. **Найти лифт и вызвать его** — запускает финальную фазу: таймер 18 секунд, монстр становится агрессивным, двери закрываются вокруг, нужно продержаться до прибытия лифта.

Опасность нарастает с прогрессом: больше тупиков, узкие коридоры, монстр быстрее, открывает двери, реагирует на свет и бег.

## Реализованные системы (архитектура)

Код разделён на независимые системы (в порядке объявления в `<script>`):

1. **Config (CFG)** — все настраиваемые параметры: разрешение рендера, FOV, скорости, яркость, громкость, масштаб, флаги постобработки.
2. **Utilities** — hash, lerp, clamp, dist, smoothstep, valueNoise, fbm (детерминированный шум для генерации).
3. **Textures** — процедурная генерация RGBA-текстур 64×64 (Uint8Array): стены, пол, потолок, 4 типа дверей + выход. Без внешних файлов.
4. **World / Chunks** — бесконечный мир на чанках 16×16, Map-хранилище, LRU-выгрузка (макс. 64 чанка) с очисткой дверей. Генерация: сетка коридоров каждые 7 клеток, комнаты 7×7 с дверями на стыках с коридорами, 9 типов спецкомнат (archive/server/storage/medical/kitchen/security/generator/lift/utility). Опасность влияет на генерацию (больше стен/тупиков).
5. **Doors** — Map дверей, 6 типов (офисная/металлическая/аварийная/деревянная/выход), анимация открытия (openAmount 0→1), запертые (нужен ключ), заклинившие, двери с выбором (аварийные — вопрос об открытии), самозахлопывание после 6 документов. Защита от дублей через chunkDoorSeeds.
6. **Items** — кольцевой спавн вокруг старта (радиус 4–30): 8 документов, 3 детали генератора, 12 батареек, 4 ключа, 6 таблеток, 3 карты, 4 фото, 3 инструмента.
7. **Documents** — 8 текстовых документов с лором и gameplay-триггерами (look_back, lights_out, hear_run, flashlight, door_slam, red_doors).
8. **Generator & Elevator** — размещаются процедурно недалеко от старта. Генератор требует 3 детали. Лифт требует питание.
9. **Objectives** — прогресс (документы/детали/питание/лифт), HUD, onProgressChanged (меняет сложность).
10. **Player** — движение, коллизии, приседание (плавный lerp), бег с выносливостью, head bob, фонарь с разрядом батареи, подбор предметов.
11. **A* Pathfinding** — поиск пути на сетке с обходом стен, двери проходимы при открытии ≥0.75, ограничение по узлам (200–350) для производительности.
12. **Monster FSM** — состояния: idle → heard → searching → near → chasing → lost. Линия видимости (hasLineOfSight), слух (зависит от бега/фонаря/приседания), память (lastKnownX/Y), патруль, прямое преследование при LOS, урон, позиционный звук. Монстр «учится»: с 3 док — реагирует на фонарь, с 4 — на бег, с 6 — открывает двери.
13. **Renderer** — raycaster DDA, текстурированные стены, потелочный/напольный кастинг по строкам, фонарь (конус света), аварийный красный свет без питания, туман, мерцание ламп с питанием. Внутренний буфер 480×270, upscaling через `image-rendering: pixelated`.
14. **Sprites** — рендер предметов, генератора, лифта, монстра с z-buffer-тестом, освещением и фонарём.
15. **Particles** — пыль/туман в воздухе (обновление + рендер как точки в 3D-проекции с z-тестом).
16. **PostProcess** — film grain (зерно плёнки), vignette (виньетка, 2×2 блоками), chromatic aberration (хроматическая аберрация). Прямая манипуляция пиксельным буфером. Включаются/выключаются в настройках.
17. **Audio** — Web Audio API: фоновый гул (2 осциллятора), шум вентиляции, позиционные звуки (шаги, рык монстра), шумовые бёрсты, тоны, окружающие звуки (капли, трубы). Громкость настраивается.
18. **Events** — случайные события каждые 3–5 мин (зависит от опасности): мерцание, открытие дверей, телефон, кровь, стуки, хлопанье дверей, звуки. Триггеры от документов.
19. **Final Phase** — таймер лифта 18с, монстр в режиме бесконечной погони, двери закрываются вокруг.
20. **Save** — autosave в localStorage на каждом milestone (документ, предмет, генератор, лифт, смерть/победа). Сохраняет позицию, прогресс, прочитанные документы, собранные предметы, состояние лифта/генератора/монстра, настройки.
21. **UI / Menu** — главное меню (Продолжить/Новая игра/Настройки/Управление), настройки (чувствительность/яркость/громкость/масштаб рендера/зерно/виньетка/хром. аберрация), HUD (документы/генератор/питание/лифт/этаж/состояние монстра + здоровье/батарея/выносливость), подсказки взаимодействия, субтитры, экран смерти/победы.
22. **Input** — keyboard/mouse, Pointer Lock, обработка паузы (ESC возобновляет из меню без перезагрузки), взаимодействие.

## Технические детали

- **Рендер**: внутренний буфер 480×270 (или масштабированный), `ImageData` + `putImageData`, `image-rendering: pixelated` для апскейла.
- **Мир**: чанки 16×16, LRU-кэш на 64 чанка, детерминированная генерация через value-noise/fBm.
- **Pathfinding**: A* на сетке, двери проходимы при открытии, ограничение узлов.
- **Сохранение**: `localStorage`, ключ `backrooms_b2_save`, JSON.
- **Аудио**: Web Audio API, осцилляторы + шумовые буферы + фильтры, позиционный звук через громкость по дистанции.
- **Постобработка**: прямая манипуляция `Uint8ClampedArray` пиксельного буфера.
